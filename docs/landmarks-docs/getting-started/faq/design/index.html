Frequently asked questions

Here you will find frequently asked questions about HDS and answers to those questions. If you do not find an answer to your question on this page, you can always contact the HDS team for more information.

Design

What design software should I use?
The official design software used in the City of Helsinki are Sketch and Abstract. These are also software that the HDS supports - meaning that the design kit is shared for the Sketch software only. It is recommended to use Sketch in your project if it is possible. It ensures that the HDS team can support you in problem situations.
Is there a HDS Figma library available?
The HDS only officially supports Sketch and Abstract. Figma is not officially supported and the team cannot provide any support for Figma issues. It is recommended to use Sketch and Abstract. Figma may be supported in the future but currently, there are no plans for it.
How can I get the Helsinki Grotesk font to my design software?
In order to use the font in a design software, you need to have the font locally on your computer. Please refer to the internal font documentation in Confluence and follow the desiger instructions. If you do not have access to Confluence, ask from your projects city employee contact.
My Abstract projects take a long time to open or save. What should I do?

Most often this is caused by large Sketch file sizes. Abstract recommends Sketch files to be below 50MB in size. If you files are larger than this, here are a couple of tips to reduce files' sizes:

  1. Reorganise your files. This means creating multiple smaller files from one larger file. If many of your files use the same components, create one file to hold those components and use that file as a library. Read more about creating library files in the Abstracts documentation.
  2. In Sketch, use File Reduce File Size feature. This uses multiple methods to reduce the file size.
  3. Check your images and compress them if needed. Prefer .png and .jpeg file formats. Images are often the largest source of file sizes in Sketch projects.

How to customize Sketch typography styles (for example, wants to use a font with a color that is not part of the default library)

Generally, only black and white typography is recommended in the City of Helsinki services. If your project needs to use other colours (e.g. brand colours) in typography, you will quickly find out that these colour variations are not available in the shared Sketch styles and Sketch does not allow you to change the text colour without breaking the link to the original style or symbol.

In this case, you need to create your text style and replace the default one with that:

  1. Create a text element and give it a style of the same size you need. E.g. if you need a heading M size text, use heading M black style.
  2. Change the colour of the inserted text.
  3. Create a new text style for this modified text. You can do this by pressing the Create button in the Appearance side menu. Give it a descriptive name, e.g. Heading M - 03 Bussi
  4. Now you have a custom text style that you can use in HDS components. Just override the text style with your style in the Overrides side menu.

Which HDS component colors can I customise?

First, please take a look at the Colour token documentation for general information about using the colours.

There are a few rules of thumb that you can follow when customising colours:

  1. If the colour is editable in the symbol override menu in Sketch, it can be customised.
  2. If the default colour is Black or White, you can replace it with a Black, a White or a Grayscale colour.
  3. If the default colour is a brand colour, you can replace it with another brand colour.

If you are in doubt, you can always ask questions in the #designsystem Slack channel or via other HDS contact methods.

Where can the icons be found in vector/PNG/etc. format?
You can download the newest HDS icon kit from GitHub. The kit contains all HDS icons in SVG, PNG, EPS and PDF formats.
